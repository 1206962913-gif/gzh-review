# 免费云部署方案

## 推荐选择

### 1. Koyeb Free

适合：先让电脑和手机都能访问，少量人员试用。

优点：
- 支持 Node.js Web 服务
- 有一个免费 Web Service
- 可以绑定自定义域名

限制：
- 免费资源较小
- 国内访问速度取决于网络情况
- 公众号链接读取是否稳定，需要实际测试

部署思路：

```text
GitHub 仓库 -> Koyeb 新建 Web Service -> Start command: node server.js
```

具体步骤：

1. 注册并登录 Koyeb。
2. 把本项目上传到 GitHub 仓库。
3. 在 Koyeb 控制台选择 `Create Service`。
4. Source 选择 `GitHub`，授权后选择项目仓库。
5. 部署类型选择 Web Service。
6. Build command 填：

```bash
npm install
```

7. Run command 填：

```bash
npm start
```

8. Health check path 填：

```text
/health
```

9. 部署完成后，Koyeb 会生成一个公网地址，例如：

```text
https://your-app.koyeb.app
```

10. 手机和电脑都访问这个地址即可。

如果要绑定正式域名，在 Koyeb 的 Domains 里添加域名，再按页面提示配置 DNS。

### 2. Render Free Web Service

适合：演示、试用、低频访问。

优点：
- 支持 Node.js Web Service
- 部署简单
- 可绑定域名

限制：
- 免费服务可能休眠，首次打开会慢
- 不适合正式高频使用

部署思路：

```text
GitHub 仓库 -> Render New Web Service -> Start command: node server.js
```

### 3. Railway Free / Trial

适合：短期测试。

优点：
- 部署体验好
- 支持 Node.js

限制：
- 免费额度较少
- 试用期和网络权限受账号验证影响
- 长期使用大概率需要付费

### 4. Oracle Cloud Always Free

适合：想要更接近“真正服务器”的免费方案。

优点：
- 有 Always Free 资源
- 可以自己安装 Node.js、Nginx、HTTPS
- 更接近正式服务器部署

限制：
- 注册门槛较高，通常需要银行卡验证
- 运维复杂度比 Koyeb / Render 高
- 可用地区和机器资源不一定随时有

## 我的建议

先用 Koyeb 或 Render 做试运行，确认：

1. 手机和电脑都能打开页面
2. 公众号正式链接能读取正文和图片
3. 临时预览链接是否能读取
4. 访问速度是否能接受

如果后续要给多个部门长期使用，再迁移到正式云服务器或单位认可的云资源。

## 项目部署参数

本项目部署时只需要：

```text
Build command: 留空或 npm install
Start command: node server.js
Port: 使用平台自动注入的 PORT
```

代码已经支持平台自动分配端口：

```text
process.env.PORT
```
