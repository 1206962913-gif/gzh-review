# 公众号保密审核网站部署说明

## 1. 服务器要求

- 安装 Node.js 20 或更高版本
- 服务器能访问互联网，用于读取 `mp.weixin.qq.com` 文章内容和图片
- 手机、电脑都能访问服务器域名或内网地址

## 2. 启动服务

进入项目目录：

```bash
cd /部署目录/公众号保密审核
npm start
```

默认访问地址：

```text
http://服务器IP:4173/
```

也可以指定端口：

```bash
PORT=8080 npm start
```

健康检查地址：

```text
http://服务器IP:4173/health
```

## 3. 正式域名访问

如果域名是：

```text
https://review.example.com
```

建议用 Nginx 做反向代理，把域名转发到本服务。

Nginx 示例：

```nginx
server {
    listen 80;
    server_name review.example.com;

    location / {
        proxy_pass http://127.0.0.1:4173;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如果正式使用，建议同时配置 HTTPS 证书。

## 4. 内网服务器访问

如果只在单位内网使用：

1. 把项目放到内网服务器
2. 启动服务：`npm start`
3. 用内网 DNS 或域名解析到服务器 IP
4. 电脑和手机访问同一个内网域名

示例：

```text
http://gzh-review.company.local/
```

## 5. 后台长期运行

可以用 `pm2` 管理服务：

```bash
npm install -g pm2
pm2 start server.js --name gzh-review
pm2 save
```

常用命令：

```bash
pm2 status
pm2 restart gzh-review
pm2 logs gzh-review
```

## 6. 注意事项

- 临时预览链接如果需要微信登录态，服务器可能无法直接读取正文。
- 如果链接已过期或权限受限，页面会提示无法解析。
- 词库文件是 `sensitive-rules.js`，后续增删敏感词优先维护这个文件。
